---
argument-hint: [phase-number]
description: 执行 plan.md 中指定 Phase 的所有任务
---

处理 plan.md 中的 Phase $ARGUMENTS：

1. **读取计划**
   - 打开 plan.md 文件
   - 定位到 Phase $ARGUMENTS 部分
   - 列出该阶段的所有待完成任务

2. **执行任务**
   - 按顺序执行每个未完成的任务
   - 实现最小可行代码
   - 添加必要的注释

3. **测试验证**
   - 运行相关的单元测试
   - 运行集成测试（如适用）
   - 确保所有测试通过

4. **更新进度**
   - 在 plan.md 中标记已完成的任务 [x]
   - 更新 progress.md 记录当前状态
   - 记录遇到的任何问题

5. **报告结果**
   - 显示该 Phase 的完成进度
   - 列出完成的任务
   - 显示测试结果摘要

6. **提示下一步**
   - 询问是否继续下一个 Phase
   - 或者是否需要提交当前代码
