基于需求 "$ARGUMENTS" 创建详细的实施计划文档（plan.md）：

## 步骤
1. **分析需求复杂度**
   - 分析 "$ARGUMENTS" 的任务规模
   - 识别主要技术栈
   - 列出依赖关系

2. **分解为阶段（Phases）**
   - 将任务分为 3-5 个主要阶段
   - 每个阶段有明确的交付物
   - 阶段之间有依赖关系

3. **细化每个阶段**
   - 列出具体的可执行步骤
   - 为每个步骤添加 Markdown 检查清单
   - 估算每个步骤的时间

4. **添加支持信息**
   - 技术决策及理由
   - 潜在风险和缓解措施
   - 完成标准

5. **创建文件**
   - 将完整计划保存到 `plan.md`
   - 使用标准的 Markdown 格式
   - 包含目录和导航链接

## 输出格式
参考 `common/templates/plan.md` 模板文件。
